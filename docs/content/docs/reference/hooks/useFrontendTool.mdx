`useFrontendTool` is a React hook that you can use in your application to provide
tools that can be called by the AI. Essentially, it allows the Copilot to
execute these tools contextually during a chat, based on the user's interactions
and needs.

Here's how it works:

Use `useFrontendTool` to set up tools that the Copilot can call. To provide
more context to the Copilot, you can provide it with a `description` (for example to explain
what the tool does, under which conditions it can be called, etc.).

Then you define the parameters of the tool using Zod schemas, which provide
runtime validation and TypeScript type inference.

Finally, you provide a `handler` function that receives the parameters and returns a result.
CopilotKit takes care of automatically inferring the parameter types, so you get type safety
and autocompletion for free.

To render a custom UI for the tool, you can provide a `render()` function. This function
lets you render a custom component or return a string to display.

## Usage

### Simple Usage

```tsx
import { useFrontendTool } from "@copilotkit/react-core";
import { z } from "zod";

useFrontendTool({
  name: "sayHello",
  description: "Say hello to someone.",
  parameters: z.object({
    name: z.string().describe("name of the person to greet"),
  }),
  handler: async ({ name }) => {
    alert(`Hello, ${name}!`);
  },
});
```

### With Custom UI Rendering

```tsx
import { useFrontendTool } from "@copilotkit/react-core";
import { z } from "zod";

useFrontendTool({
  name: "showWeatherCard", 
  description: "Display weather information for a location",
  parameters: z.object({
    location: z.string().describe("The location to show weather for"),
    temperature: z.number().describe("Temperature in celsius"),
  }),
  handler: async ({ location, temperature }) => {
    // Fetch and return weather data
    return { location, temperature, conditions: "Sunny" };
  },
  render: ({ args, status, result }) => {
    if (status === "inProgress") {
      return <div>Loading weather for {args.location}...</div>;
    }
    if (status === "complete" && result) {
      return (
        <WeatherCard 
          location={result.location}
          temperature={result.temperature}
          conditions={result.conditions}
        />
      );
    }
    return null;
  },
});
```

## Generative UI

This hook enables you to dynamically generate UI elements and render them in the copilot chat. For more information, check out the [Generative UI](/guides/generative-ui) page.

## Parameters

<PropertyReference name="tool" type="ReactFrontendTool<T>" required>
  The tool configuration object.

    <PropertyReference name="name" type="string" required>
      The name of the tool.
    </PropertyReference>

    <PropertyReference name="handler" type="(args: T) => Promise<any>">
      The handler function that executes the tool logic.
    </PropertyReference>

    <PropertyReference name="description" type="string">
      A description of the tool. This is used to instruct the Copilot on how to
      use the tool.
    </PropertyReference>

    <PropertyReference name="followUp" type="boolean" default="true">
        Whether to report the result of the tool call to the LLM which will then provide a follow-up response. Pass `false` to disable.
    </PropertyReference>

    <PropertyReference name="parameters" type="z.ZodType<T>">
      The Zod schema defining the parameters of the tool. This provides runtime validation
      and TypeScript type inference.
    </PropertyReference>

    <PropertyReference name="render" type="React.ComponentType<RenderProps>">
      A React component that renders custom UI for the tool. The component receives props with:
      
      <div className="ml-8">
        <PropertyReference name="status" type="'inProgress' | 'executing' | 'complete'">
          - `"inProgress"`: arguments are dynamically streamed to the function, allowing you to adjust your UI in real-time.
          - `"executing"`: The tool handler is executing.
          - `"complete"`: The tool handler has completed execution.
        </PropertyReference>

        <PropertyReference name="args" type="Partial<T> | T">
          The arguments passed to the tool in real time. When the status is `"inProgress"`, they are
          possibly incomplete.
        </PropertyReference>

        <PropertyReference name="result" type="any">
          The result returned by the tool. It is only available when the status is `"complete"`.
        </PropertyReference>

        <PropertyReference name="name" type="string">
          The name of the tool being executed.
        </PropertyReference>

        <PropertyReference name="description" type="string">
          The description of the tool being executed.
        </PropertyReference>
      </div>
    </PropertyReference>
</PropertyReference>

<PropertyReference name="dependencies" type="any[]">
  An optional array of dependencies.
</PropertyReference>

## Migration from useCopilotAction

If you're migrating from `useCopilotAction`, here are the key differences:

1. **Parameters use Zod schemas** instead of the parameter array format
2. **Type inference is automatic** from the Zod schema
3. **The render component props** include `name` and `description`

### Migration Example

```tsx
// Before with useCopilotAction
useCopilotAction({
  name: "addTodo",
  parameters: [
    {
      name: "text",
      type: "string",
      description: "The todo text",
      required: true,
    },
  ],
  handler: ({ text }) => {
    addTodo(text);
  },
});

// After with useFrontendTool
useFrontendTool({
  name: "addTodo",
  parameters: z.object({
    text: z.string().describe("The todo text"),
  }),
  handler: ({ text }) => {
    addTodo(text);
  },
});
```
