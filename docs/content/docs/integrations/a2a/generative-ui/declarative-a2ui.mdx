## Getting started

<Steps>
  <Step>
    ### Scaffold the starter

    ```bash
    npx copilotkit@latest create my-app --framework a2ui
    cd my-app
    ```

    The `a2ui` framework scaffolds a Next.js app paired with a LangGraph Python agent, preconfigured with an A2UI catalog and the middleware wiring. `-f a2ui` works as a shorthand.
  </Step>

  <Step>
    ### Install and run

    ```bash
    pnpm install
    cp .env.example .env   # add your OPENAI_API_KEY
    pnpm dev
    ```

    `pnpm dev` starts the Next.js UI and the LangGraph agent concurrently. Open http://localhost:3000 and send a message — the agent will respond with an A2UI surface rendered from your catalog.
  </Step>

  <Step>
    ### How A2UI works

    A2UI has three moving parts:

    - **Catalog** — a set of component definitions (Zod schemas) paired with React renderers. Registered once on the frontend.
    - **Surface** — a rendered UI instance. The agent creates a surface against a catalog, then updates its components and data.
    - **Operations** — `createSurface`, `updateComponents`, `updateDataModel`. The middleware streams these to the client.

    There are two patterns for where the component tree comes from:

    | Pattern | Component tree | Best for |
    |---|---|---|
    | **Fixed schema** | Pre-defined in a JSON file; agent provides data only | Known layouts (flight cards, product lists) |
    | **Dynamic schema** | A secondary LLM generates the tree and data per turn | Open-ended UI that changes with the conversation |

    Both patterns share the same frontend catalog.
  </Step>

  <Step>
    ### Register the catalog in the app

    The catalog is registered once, at the root of the tree:

    ```tsx title="apps/app/src/app/layout.tsx"
    "use client";
    import "@copilotkit/react-core/v2/styles.css";
    import { CopilotKit } from "@copilotkit/react-core/v2";
    import { demonstrationCatalog } from "./declarative-generative-ui/renderers";

    export default function RootLayout({
      children,
    }: {
      children: React.ReactNode;
    }) {
      return (
        <html lang="en">
          <body>
            <CopilotKit
              runtimeUrl="/api/copilotkit"
              a2ui={{ catalog: demonstrationCatalog }}
            >
              {children}
            </CopilotKit>
          </body>
        </html>
      );
    }
    ```

    <Callout type="info">
    The import path is `@copilotkit/react-core/v2` — A2UI surfaces render through the v2 runtime.
    </Callout>
  </Step>

  <Step>
    ### Define a custom catalog

    A catalog has two files: **definitions** (what the agent sees) and **renderers** (what the user sees). Both live under `apps/app/src/app/declarative-generative-ui/` in the starter.

    **Definitions** — Zod schemas plus a description. The description is how the agent learns what each component is for:

    ```ts title="apps/app/src/app/declarative-generative-ui/definitions.ts"
    import { z } from "zod";

    export const demonstrationCatalogDefinitions = {
      Metric: {
        description:
          "A key metric display with label, value, and optional trend indicator. Great for KPIs and stats.",
        props: z.object({
          label: z.string(),
          value: z.string(),
          trend: z.enum(["up", "down", "neutral"]).optional(),
          trendValue: z.string().optional(),
        }),
      },

      DashboardCard: {
        description:
          "A card container with title and optional subtitle. Has a 'child' slot for content.",
        props: z.object({
          title: z.string(),
          subtitle: z.string().optional(),
          child: z.string().optional(),
        }),
      },
    };

    export type DemonstrationCatalogDefinitions =
      typeof demonstrationCatalogDefinitions;
    ```

    **Renderers** — React components, type-checked against the schemas:

    ```tsx title="apps/app/src/app/declarative-generative-ui/renderers.tsx"
    "use client";
    import {
      createCatalog,
      type CatalogRenderers,
    } from "@copilotkit/a2ui-renderer";
    import {
      demonstrationCatalogDefinitions,
      type DemonstrationCatalogDefinitions,
    } from "./definitions";

    const renderers: CatalogRenderers<DemonstrationCatalogDefinitions> = {
      Metric: ({ props }) => (
        <div>
          <span>{props.label}</span>
          <strong>{props.value}</strong>
          {props.trend && <span>{props.trendValue}</span>}
        </div>
      ),

      DashboardCard: ({ props, children }) => (
        <div className="rounded-xl border p-5">
          <h3>{props.title}</h3>
          {props.subtitle && <p>{props.subtitle}</p>}
          {props.child && children(props.child)}
        </div>
      ),
    };

    export const demonstrationCatalog = createCatalog(
      demonstrationCatalogDefinitions,
      renderers,
      { catalogId: "copilotkit://app-dashboard-catalog" },
    );
    ```

    <Callout type="info">
    The `catalogId` is how your agent references this catalog. Use a stable URI-style string that's unique to your app.
    </Callout>
  </Step>

  <Step>
    ### Point your agent at the catalog

    On the agent side, every surface is created against a `catalog_id`:

    ```python title="apps/agent/src/a2ui_dynamic_schema.py"
    from copilotkit import a2ui

    CUSTOM_CATALOG_ID = "copilotkit://app-dashboard-catalog"

    ops = [
        a2ui.create_surface(surface_id, catalog_id=CUSTOM_CATALOG_ID),
        a2ui.update_components(surface_id, components),
        a2ui.update_data_model(surface_id, data),
    ]
    return a2ui.render(operations=ops)
    ```

    The same pattern applies for fixed-schema tools — the only difference is that `components` is loaded from a JSON file with `a2ui.load_schema(...)` instead of being generated by an LLM.

    For deeper coverage of each pattern, see the LangGraph guides:
    - [Fixed Schema](/integrations/langgraph/generative-ui/a2ui/fixed-schema)
    - [Dynamic Schema](/integrations/langgraph/generative-ui/a2ui/dynamic-schema)
  </Step>

  <Step>
    ### Design components with the A2UI Composer

    To design and preview A2UI components visually, use the [A2UI Composer](https://a2ui-composer.ag-ui.com/) — built by CopilotKit engineers and updated for v0.9.

    - `/basic-catalog` documents the standard A2UI v0.9 components.
    - `/custom-catalog` shows a worked example of a custom catalog you can riff on.
    - `/gallery` has reference widgets you can open and edit.

    The editor produces v0.9 JSON which you can copy straight into your agent (for fixed schemas) or hand to a secondary LLM (for dynamic schemas).
