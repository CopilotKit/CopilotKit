## What is typegen?

Typegen is a small CLI that introspects your running CopilotKit runtime and emits a TypeScript declaration file. Once generated, every `agentId` prop in `@copilotkit/react-core/v2` is constrained to the agents your runtime actually exposes — autocomplete works, typos fail type-check.

```bash
npx copilotkit typegen http://localhost:3000/api/copilotkit
```

Without typegen, `agentId` is just `string`. With typegen, it becomes a union of the agent IDs your runtime registered.

<Callout type="info">
  Typegen currently only constrains **agent IDs**. Tool names registered via
  `useFrontendTool`, `useRenderTool`, and friends remain unconstrained — you
  are free to register any client-side tool name, regardless of what the
  runtime exposes.
</Callout>

## How it works

```text
Your Runtime (running server)
  |  GET /info
  v
copilotkit typegen <url>
  |  reads agent names
  v
.copilotkit/types.ts       — type Agents = { ... }
.copilotkit/register.d.ts  — augments Register interface
  |
  v
TypeScript picks up the augmentation; AgentId resolves to your agent union.
```

### Step 1 — Runtime exposes agent info

Every CopilotKit runtime ships with a `/info` endpoint that returns the agents registered on the server, including their name and description.

### Step 2 — CLI generates types

Running `copilotkit typegen` writes two files into `.copilotkit/` at your project root:

#### `.copilotkit/types.ts`

```ts
// Auto-generated by @copilotkit/cli typegen
// Do not edit manually

export type Agents = {
  "default": {
    description: "Default conversational agent";
  };
  "planner": {
    description: "Plans and schedules events";
  };
};
```

#### `.copilotkit/register.d.ts`

```ts
import type { Agents } from "./types";

declare module "@copilotkit/react-core/v2" {
  interface Register {
    agents: Agents;
  }
}
```

The CLI also automatically:

- Adds `.copilotkit/` to your `.gitignore` (generated files should not be committed)
- Adds `.copilotkit/register.d.ts` to your `tsconfig.json` `include` array

### Step 3 — Hooks pick up the augmentation

```tsx
import { useAgent, useFrontendTool } from "@copilotkit/react-core/v2";

// agentId is now "default" | "planner" — autocomplete + compile-time check
const { agent } = useAgent({ agentId: "planner" });

// Tool name stays unconstrained — register any client-side tool you like
useFrontendTool({
  name: "anything",
  description: "Client-side only — runtime doesn't need to know",
  agentId: "default", // <- still type-checked
  handler: async () => {},
});
```

## Multiple runtimes

Pass a comma-separated list of URLs to merge agent IDs from multiple runtimes:

```bash
npx copilotkit typegen http://localhost:3000/api/copilotkit,http://localhost:4000/api/copilotkit
```

If the same agent ID is exposed by multiple runtimes, the last one wins (with a warning logged).

## When to re-run

Re-run `copilotkit typegen` whenever you add, remove, or rename an agent on the server. The generated `register.d.ts` is small and deterministic — drop it in CI if you want to enforce that everyone's local types match the deployed runtime.
