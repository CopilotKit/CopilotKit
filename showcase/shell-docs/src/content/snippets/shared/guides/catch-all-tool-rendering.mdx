## Customize the catch-all

Once you want on-brand chrome, pass a `render` function to
`useDefaultRenderTool`. It's a convenience wrapper around
`useRenderTool({ name: "*", ... })`: one wildcard renderer handles every tool
call not claimed by a named renderer.

## Render nothing from the catch-all

The catch-all can also be a deliberate opt-out. Return `null` when you want
otherwise-unhandled tool calls to run without painting UI in the chat.

Include `status` in the signature when the renderer may later become
conditional:

```tsx title="app/page.tsx"
import { useDefaultRenderTool } from "@copilotkit/react-core/v2"; // [!code highlight]

function YourMainContent() {
  useDefaultRenderTool({
    render: ({ status }) => null,
  });

  return <ChatInterface />;
}
```

You can make the opt-out conditional too. For example, return `null` while
`status` is in progress, then render a result card when the call completes.

If you are configuring renderers at the provider boundary, register the same
wildcard explicitly:

```tsx title="app/page.tsx"
import {
  CopilotKit,
  defineToolCallRenderer,
} from "@copilotkit/react-core/v2";

const suppressUnhandledToolCalls = defineToolCallRenderer({
  name: "*",
  render: ({ status }) => null,
});

function App() {
  return (
    <CopilotKit
      runtimeUrl="/api/copilotkit"
      renderToolCalls={[suppressUnhandledToolCalls]}
    >
      <YourApp />
    </CopilotKit>
  );
}
```

## Suppress one named frontend tool

For a single frontend tool, return `null` from that tool's renderer. A named
renderer takes priority over the catch-all, even when that named renderer
returns nothing.

```tsx title="app/page.tsx"
import { useFrontendTool } from "@copilotkit/react-core/v2";
import { z } from "zod";

function App() {
  useFrontendTool(
    {
      name: "searchDatabase",
      parameters: z.object({ query: z.string() }),
      handler: async ({ query }) => {
        /* ... */
      },
      render: ({ status }) => null,
    },
    [],
  );

  return <ChatInterface />;
}
```
