## What is this?
LangGraph agents can receive runtime configuration from your frontend, such as user preferences, session metadata, or authentication tokens. With LangGraph 0.6.0+, the recommended approach is using **`useAgentContext`** from CopilotKit's v2 API.

If you wish to read further, you can refer to [the configuration guide by LangGraph](https://docs.langchain.com/oss/python/langgraph/use-graph-api#add-runtime-configuration)

<Callout type="warning" title="LangGraph 0.6.0+ Compatibility">
  LangGraph 0.6.0 introduced runtime `context` as the preferred mechanism for passing configuration, replacing the older `configurable` pattern. If you're using LangGraph 0.6.0+, use the **`useAgentContext`** approach shown below. The legacy `forwardedProps.config.configurable` pattern is no longer compatible with modern LangGraph versions.
</Callout>

## When should I use this?

This is useful when you want to send execution-time configuration information (such as user preferences, tokens, or metadata for a given session) that should not be part of the agent state but should influence how the agent behaves.

<Callout type="info" title="Looking for authentication?">
  If you need to pass authentication tokens specifically, see the dedicated [Authentication guide](/langgraph/auth) which covers LangGraph Platform and self-hosted authentication patterns in detail.
</Callout>

## Implementation (v2 - Recommended)

The recommended approach uses **`useAgentContext`** from `@copilotkit/react-core/v2`. This pattern works seamlessly with LangGraph 0.6.0+ and the `CopilotKitMiddleware`.

<Steps>
<Step>
### Publish context from the frontend

Use the `useAgentContext` hook to publish configuration values that your agent can access:

```tsx title="app/page.tsx"
import { useAgentContext } from "@copilotkit/react-core/v2"; // [!code highlight]

function YourMainContent() {
  // Publish configuration context that the agent can access
  // [!code highlight:8]
  useAgentContext({
    description: "User preferences and session configuration",
    value: {
      authToken: 'example-token',
      userPreference: 'detailed',
      sessionId: 'abc123'
    },
  });

  return (... your component UI)
}
```

<Callout type="tip" title="Dynamic configuration">
  The context value updates automatically when your state changes. For example, if `authToken` comes from a React state variable, the agent receives the updated value on subsequent runs.
</Callout>
</Step>
<Step>
### Set up the middleware in your agent

Add `CopilotKitMiddleware` to your LangGraph agent. The middleware automatically injects the context into your agent's execution:

```python title="agent.py"
from langgraph.prebuilt import create_agent
from langchain_openai import ChatOpenAI
from copilotkit import CopilotKitMiddleware  # [!code highlight]

graph = create_agent(
    model=ChatOpenAI(model="gpt-4"),
    tools=[],
    middleware=[CopilotKitMiddleware()],  # [!code highlight]
)
```
</Step>
<Step>
### Access context in your agent nodes

The context is available through the standard LangGraph `config` parameter. The middleware makes it accessible via `config["context"]` (LangGraph 0.6.0+) with automatic fallback to `config["configurable"]` for older versions:

```python title="agent_node.py"
from langchain_core.runnables import RunnableConfig
from typing import Any

async def agent_node(state: AgentState, config: RunnableConfig) -> AgentState:
    # Access the context published from useAgentContext
    context = config.get("context", {}) or config.get("configurable", {})
    
    auth_token = context.get("authToken")
    user_preference = context.get("userPreference")
    session_id = context.get("sessionId")
    
    # Use the context values in your agent logic
    if auth_token:
        # Make authenticated requests
        user_data = fetch_user_data(auth_token)
    
    return state
```

<Callout type="info" title="Compatibility">
  The `CopilotKitMiddleware` handles both LangGraph 0.6.0+ (`context`) and older versions (`configurable`) automatically, so your agent code works across versions.
</Callout>
</Step>
</Steps>

## Legacy Pattern (LangGraph <0.6.0)

<details>
<summary>Click to expand legacy pattern for LangGraph versions before 0.6.0</summary>

<Callout type="warning" title="Deprecated for LangGraph 0.6.0+">
  This pattern is **not compatible** with LangGraph 0.6.0 and later. LangGraph 0.6.0 introduced runtime `context` and explicitly rejects requests with both `configurable` and `context` set. Use the v2 `useAgentContext` pattern above instead.
</Callout>

### Legacy: Pass via forwardedProps.config

```tsx title="app/page.tsx (legacy)"
import { useAgent } from "@copilotkit/react-core/v2";
import { useEffect } from "react";

function YourMainContent() {
  const { agent } = useAgent({
    agentId: "sample_agent",
  });

  useEffect(() => {
    agent.runAgent({
      forwardedProps: {
        config: {
          configurable: {
            authToken: 'example-token'
          },
          recursion_limit: 50,
        }
      }
    });
  }, []);

  return (...);
}
```

### Legacy: Access in agent node

```python title="agent_node.py (legacy)"
async def agent_node(state: AgentState, config: RunnableConfig):
    auth_token = config['configurable'].get('authToken', None)
    return state
```

**Known issues with this pattern:**
- Does not work with LangGraph 0.6.0+ (HTTP 400 error: "Cannot specify both configurable and context")
- Configuration may become `None` on subsequent agent runs in the same session
- Not compatible with `@ag-ui/langgraph` 0.0.31+

</details>
