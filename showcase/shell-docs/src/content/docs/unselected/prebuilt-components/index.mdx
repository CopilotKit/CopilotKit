## Pre-built components for agentic chat

CopilotKit ships three prebuilt chat components that connect directly to your LangGraph agent. Each is a wrapper around `CopilotChat` with a different layout — pick the one that fits your app and you're done. They handle streaming, generative UI, and deep customization, so you can focus on your agent's behavior, not UI plumbing.

```tsx title="layout.tsx"

```

## The customization ladder

CopilotKit's design principle: **you should never have to throw the prebuilt UI away** to get the look you want. Step down only when you need more control.

- **Level 1 — Drop in as-is** — the components above, no configuration.
- **Level 2 — [CSS customization](./custom-look-and-feel/css)** — override theme tokens (`--copilot-kit-primary-color`, etc.) or `.copilotKit...` classes.
- **Level 3 — [Slots (subcomponents)](./custom-look-and-feel/slots)** — swap message bubble, composer, welcome screen, header, or toggle button with your own React component.
- **Level 4 — [Fully headless](./custom-look-and-feel/headless-ui)** — compose your own chat from the low-level hooks (`useAgent`, `useCopilotKit`, `useRenderToolCall`).

You can mix all four levels in the same app — change one color token, replace one slot, and keep everything else prebuilt.

## Deep customization

Start with defaults, customize as you need. All three components support the [slot system](./custom-look-and-feel/slots) — from Tailwind classes to full component replacement. See the [Slots guide](./custom-look-and-feel/slots) for runnable examples.

What you can override:

- **Labels** — swap any string via the `labels` prop
- **Slots** — override any sub-component with Tailwind classes, props, or a full replacement. See [Slots](./custom-look-and-feel/slots).
- **Fully headless** — if slots aren't enough, drop down to [`useCopilotChatHeadless_c`](./custom-look-and-feel/headless-ui) and render everything yourself.
