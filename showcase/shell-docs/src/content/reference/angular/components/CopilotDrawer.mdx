## Overview

`CopilotDrawer` is a prebuilt thread switcher for Angular. Rendered next to [`CopilotChat`](/reference/angular/components/CopilotChat), it lists the user's conversations and lets them switch, start, rename, archive, and delete threads — with **no active-thread state to wire up**. It is a thin standalone component wrapping the framework-agnostic `<copilotkit-drawer>` shadow-DOM web component, fed by `injectThreads` and the `CopilotChatConfiguration` service.

This is the Angular equivalent of React's `<CopilotDrawer>`. When `onThreadSelect` / `onNewThread` are omitted, the drawer drives the chat configuration directly: selecting a row connects the chat to that thread (and replays its history); "+ New" resets the chat to a fresh welcome screen.

## Usage

`CopilotDrawer` is a standalone component with the selector `copilot-drawer`. Import the class into your component's `imports`, provide the chat configuration so the drawer and chat share an active thread, and place the element next to `<copilot-chat>`.

```ts title="src/app/drawer-page.component.ts"
import { Component } from "@angular/core";
import {
  CopilotChat,
  CopilotDrawer,
  provideCopilotChatConfiguration,
} from "@copilotkit/angular";

@Component({
  selector: "drawer-page",
  standalone: true,
  imports: [CopilotChat, CopilotDrawer],
  providers: [provideCopilotChatConfiguration()],
  template: `
    <copilot-drawer />
    <copilot-chat />
  `,
})
export class DrawerPageComponent {}
```

Remember to import the stylesheet once in your global styles:

```css title="src/styles.css"
@import "@copilotkit/angular/styles.css";
```

## Inputs

<PropertyReference name="agentId" type="string">
  The agent whose threads to list and manage. Defaults to the agent of the surrounding chat configuration, or the platform default when none is set.
</PropertyReference>

<PropertyReference name="label" type="string" default="Threads">
  Accessible label for the drawer region and the default header text (shown when no `header` slot content is projected). Sets the element's `aria-label`.
</PropertyReference>

<PropertyReference name="onThreadSelect" type="(threadId: string) => void">
  Optional escape-hatch callback called when the user picks a thread row. When omitted, the drawer drives the chat configuration directly (`setActiveThreadId`), so a bare `<copilot-drawer>` switches the rendered thread with no host wiring.
</PropertyReference>

<PropertyReference name="onNewThread" type="() => void">
  Optional escape-hatch callback called when the user starts a new thread (the "+ New" button). When omitted, the drawer resets the chat configuration to a fresh, non-explicit thread so the welcome screen shows with no host wiring.
</PropertyReference>

<PropertyReference name="limit" type="number">
  Page size for thread pagination. When set, threads load in pages of this size and a "Load more" control appears at the bottom of the list while more remain. Omit to load the full list at once with no control.
</PropertyReference>

<PropertyReference name="data-testid" type="string" default="copilot-drawer">
  `data-testid` set on the underlying custom element, for tests and for targeting from a host page.
</PropertyReference>

## Outputs

`CopilotDrawer` does not declare its own outputs. The element's intents (select, new, archive, unarchive, delete, filter-change, retry) are handled internally and routed to `injectThreads` mutations and the `CopilotChatConfiguration` service. Use the optional `onThreadSelect` / `onNewThread` inputs to intercept the two active-thread intents.

The Angular drawer is **always licensed** (the Angular SDK is ungated), so it does not surface the element's `upsell` event.

## Customization

### Per-row content

Provide a template marked with the `copilotDrawerRow` directive to replace a row's content. It is projected into the element's `slot="row:{id}"`, so the element keeps the row chrome (selection, archived styling, actions) around your content. The thread is available as the implicit template context.

```html
<copilot-drawer>
  <ng-template copilotDrawerRow let-thread>
    <em>{{ thread.name ?? "New conversation" }}</em>
  </ng-template>
</copilot-drawer>
```

### Slots

Projected children with a `slot` attribute replace a region (`header`, `empty`, `footer`, `upsell`, `memories`, `launcher-icon`):

```html
<copilot-drawer>
  <span slot="header">My conversations</span>
</copilot-drawer>
```

### Theming

The drawer ships self-contained, theme-inheriting styles. Override the `--cpk-drawer-*` CSS variable tokens or style structural `::part()`s to restyle it; see the [React reference](/reference/components/CopilotDrawer#customization) for the full token and part list (the element is shared).
