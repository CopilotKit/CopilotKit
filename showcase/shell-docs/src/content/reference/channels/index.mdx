The channel stack turns any [AG-UI](https://docs.ag-ui.com) agent into a chat-platform bot. Three packages, three jobs:

| Package | Role |
|---------|------|
| `@copilotkit/channels` | The platform-agnostic engine: [`createBot`](/reference/channels/functions/createBot), handler registration, the agent run/tool/interrupt loop, and action binding. Re-exports the `@copilotkit/channels-ui` vocabulary. |
| `@copilotkit/channels-ui` | A pure JSX runtime + cross-platform [component vocabulary](/reference/channels/components/Message) for rich messages — no React. |
| `@copilotkit/channels-slack` | The Slack adapter: Socket Mode ingress, JSX → Block Kit via [`slack()`](/reference/channels/slack), `chat.update` streaming. |
| `@copilotkit/channels-discord` | Discord PlatformAdapter for `@copilotkit/channels`: Gateway ingress via discord.js, Components V2 egress, streaming replies, and slash-command registration. |

```sh
pnpm add @copilotkit/channels @copilotkit/channels-ui @copilotkit/channels-slack
# or, for Discord:
pnpm add @copilotkit/channels @copilotkit/channels-ui @copilotkit/channels-discord
```

## Setup

The packages ship as **ES modules only** (`"type": "module"` required). To author messages as JSX, point the TypeScript JSX factory at `@copilotkit/channels-ui` and write `.tsx` files:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "jsxImportSource": "@copilotkit/channels-ui"
  }
}
```

The package ships its own `JSX` namespace via `@copilotkit/channels-ui/jsx-runtime`, so JSX is statically type-checked: unknown attributes, wrong prop values, and bad children are compile-time errors.

## How the pieces fit

A turn flows through three stages. The adapter normalizes a platform event into a turn and hands it to your handler, which calls [`thread.runAgent()`](/reference/channels/classes/Thread). The engine drives the run loop — streaming text into the thread, executing [tools](/reference/channels/functions/defineBotTool) when the agent calls them, capturing interrupts. Anything you post is JSX lowered to the [BotNode](/reference/channels/types/BotNode) intermediate representation (plain serializable data), which the adapter translates to the platform's native format — with interactive handlers bound through the [ActionStore](/reference/channels/types/ActionStore).
