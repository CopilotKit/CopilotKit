## Overview

`renderComponents` translates the [BotNode IR](/reference/channels/types/BotNode) into a single top-level Components V2 `ContainerBuilder`; `renderDiscordMessage` additionally wraps it in the `{ components, flags }` shape required by `channel.send` / `message.edit`, with `MessageFlags.IsComponentsV2` set. The adapter calls these for you on every [`thread.post`](/reference/channels/classes/Thread) — reach for them directly when testing components or building custom egress.

## Signature

```ts
import { renderComponents, renderDiscordMessage, DISCORD_LIMITS } from "@copilotkit/channels-discord";

function renderComponents(ir: BotNode[]): ContainerBuilder;

function renderDiscordMessage(ir: BotNode[]): {
  components: ContainerBuilder[];
  flags: number; // MessageFlags.IsComponentsV2
};
```

## Component mapping

| Component | Components V2 |
|-----------|---------------|
| [`Message`](/reference/channels/components/Message) | outer `ContainerBuilder` (accent color if `<Message accent="#hex">`) |
| [`Header`](/reference/channels/components/Header) | `TextDisplayBuilder` with `# ` prefix |
| [`Section`](/reference/channels/components/Section) / [`Markdown`](/reference/channels/components/Markdown) / [`Text`](/reference/channels/components/Text) | `TextDisplayBuilder` (Markdown rendered via `discordMarkdown`) |
| [`Fields`](/reference/channels/components/Fields) | `TextDisplayBuilder` with bold-label lines (`**label** value`) |
| [`Context`](/reference/channels/components/Context) | `TextDisplayBuilder` with `-# ` subtext lines |
| [`Actions`](/reference/channels/components/Actions) | one or more `ActionRowBuilder` (buttons chunked ≤5/row, selects each get their own row) |
| [`Button`](/reference/channels/components/Button) | `ButtonBuilder` inside an `ActionRowBuilder` |
| [`Select`](/reference/channels/components/Select) | `StringSelectMenuBuilder` inside its own `ActionRowBuilder` |
| [`Image`](/reference/channels/components/Image) | `MediaGalleryBuilder` with one `MediaGalleryItemBuilder` |
| [`Divider`](/reference/channels/components/Divider) | `SeparatorBuilder` (small spacing, divider line) |
| [`Table`](/reference/channels/components/Table) | `TextDisplayBuilder` with a reconstructed GFM pipe table fenced by `discordMarkdown` |
| [`Input`](/reference/channels/components/Input) | skipped — modal-only on Discord; a warning is logged |

## IsComponentsV2 flag

`renderDiscordMessage` always sets `MessageFlags.IsComponentsV2` on the returned `flags`. Discord requires this flag when the message body contains a top-level `ContainerBuilder`; without it the API rejects the payload.

## Per-element budgets

Discord caps every element. The renderer **degrades instead of failing**: over-long text is truncated with an overflow marker (`…`), and sub-collections (select options, action rows, buttons per row) are clamped to their caps. Nothing ever exceeds a platform limit. The limits ship as `DISCORD_LIMITS` (see [DISCORD_LIMITS](/reference/channels/discord/DISCORD_LIMITS)).
