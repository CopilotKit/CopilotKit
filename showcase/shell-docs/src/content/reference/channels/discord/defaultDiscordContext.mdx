## Overview

`defaultDiscordContext` is the list of Discord-specific [`ContextEntry`](/reference/channels/functions/createBot) values the package ships, meant to be spread into [`createBot({ context })`](/reference/channels/functions/createBot). It is **not auto-applied** — you spread it explicitly. Each entry is also exported individually:

| Entry | Teaches the agent |
|-------|-------------------|
| `discordTaggingContext` | How to @-mention people: resolve via the [`lookup_discord_user`](/reference/channels/discord/defaultDiscordTools) tool and paste the returned `<@id>` mention verbatim; mention channels with `<#CHANNEL_ID>` |
| `discordFormattingContext` | What formatting survives in Discord — standard Markdown (**bold**, *italic*, `code`, fenced blocks, headings, quotes) works, but Discord renders **no tables** — put tabular data in a fenced code block or call the appropriate render tool instead |
| `discordConversationModelContext` | The Discord conversation model — each channel, thread, and DM is one conversation; the bot replies in the same channel the message arrived in, and uses the read-history tool to see prior messages when it needs earlier context |

## Usage

```ts
import { defaultDiscordContext } from "@copilotkit/channels-discord";

const bot = createBot({
  // …
  context: [...defaultDiscordContext, ...myAppContext],
});
```

Context entries are folded into the agent's system context on every [`thread.runAgent()`](/reference/channels/classes/Thread); per-run additions go through `runAgent({ context })`.
