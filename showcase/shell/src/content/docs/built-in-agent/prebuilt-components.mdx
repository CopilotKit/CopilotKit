## Variants

All three of `CopilotChat`, `CopilotSidebar`, and `CopilotPopup` share the same underlying primitives — they differ only in layout:

- **`CopilotChat`** — an inline, full-surface chat. Good for dedicated chat pages.
- **`CopilotSidebar`** — a docked sidebar that pushes content. Good for "co-present" layouts.
- **`CopilotPopup`** — a floating bubble + modal. Good for contextual copilots on landing pages or dashboards.

They all accept the same `labels`, slot props, `welcomeScreen`, `messageView`, and `input` props — swap components without rewriting anything.

## Deep customization

Start with defaults, customize as you need:

- **Labels** — swap any string via the `labels` prop
- **Slots** — override any sub-component with Tailwind classes, props, or a full replacement. See [Slots](./custom-look-and-feel/slots).
- **Fully headless** — if slots aren't enough, drop down to [`useCopilotChatHeadless_c`](./custom-look-and-feel/headless-ui) and render everything yourself.
