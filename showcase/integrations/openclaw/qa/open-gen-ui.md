# QA: Open-Ended Generative UI (minimal) (OpenClaw)

Demo source: `src/app/demos/open-gen-ui/page.tsx`
Route: `/demos/open-gen-ui` · Agent: `open-gen-ui` · Runtime: `/api/copilotkit-ogui`
Run against the real backend at `http://localhost:3119/demos/open-gen-ui`.

Status: **supported** (see `PARITY_NOTES.md`, "Tools & generative UI"). Behaviour
lives in the frontend + CopilotKit runtime middleware; the gateway is a
pass-through.

## What it exercises

The minimal Open Generative UI setup: a plain `<CopilotChat>` with no custom
tool renderers. The `open-gen-ui` runtime (`api/copilotkit-ogui/route.ts`) sets
`openGenerativeUI.agents: ["open-gen-ui", ...]`, which injects a
`generateSandboxedUi` tool. The Claude pass-through agent (via the OpenClaw
gateway) calls it, and the runtime middleware converts each call into an
`open-generative-ui` activity event that the built-in
`OpenGenerativeUIActivityRenderer` mounts as a **sandboxed iframe** inside the
chat transcript.

Because OpenClaw is a single stateless gateway with no per-demo backend, there
is **no backend rendering prompt**. The page instead swaps in a
visualisation-tuned design skill frontend-side via
`openGenerativeUI.designSkill` (`VISUALIZATION_DESIGN_SKILL` in
`./design-skill.ts`), which rides to the model as agent context and steers it
toward SVG/canvas educational figures (inline SVG, labelled axes, a legend, a
title/subtitle, CSS `@keyframes` animation, the indigo/emerald/amber/rose/slate
palette).

## Prerequisites

- Stack is up; demo reachable at the URL above.
- Gateway is healthy (per-demo agent names all map to the one OpenClaw endpoint).
- The `open-gen-ui` agent is registered in the OGUI runtime with
  `openGenerativeUI.agents` including `"open-gen-ui"`.

## Manual steps

1. Open the demo. Confirm `<CopilotChat>` renders full-height inside the centred
   `max-w-4xl` container and the input composer is visible.
2. Confirm the four suggestion pills are visible: **3D axis visualization**,
   **How a neural network works**, **Quicksort visualization**, and
   **Fourier: square wave from sines**.
3. Send a basic message (e.g. **"Hi"**). Expect the agent to call
   `generateSandboxedUi` and mount a sandboxed iframe in the assistant turn.
   Short placeholder lines (e.g. "Sketching the scene…", "Labelling axes…")
   appear while streaming, before the final UI renders.
4. In a fresh turn, click **How a neural network works**. Inside the iframe,
   confirm an inline `<svg>` renders (not stacked `<div>`s) with three layers of
   nodes and connecting lines, layer labels, and activations that pulse forward
   from input to output on a continuous loop with no user interaction.
5. In a fresh turn, click **Quicksort visualization**. Confirm a new iframe with
   ~10 SVG rect bars and a caption that updates as the sort progresses (pivot
   highlighted amber `#f59e0b`, compared elements indigo `#6366f1`).

## Assertion bar

- Each prompt produces exactly one `generateSandboxedUi` tool call and one
  mounted sandboxed iframe (no duplicate render).
- Rendered content is SVG/canvas-based with labelled axes/layers and a legend —
  it respects the design-skill palette (indigo accent, emerald success, amber
  warning, slate neutrals) and has a title + subtitle line.
- The outer card has a white background, rounded corners, and padding — content
  does not bleed to the viewport edges.
- Animations loop smoothly via CSS keyframes; the scene is self-running.
- No console errors originate from the host page (sandbox-internal warnings may
  be present). The iframe makes no network requests (no fetch/XHR/localStorage).

## Caveats

- The visualisation is generated by the model at runtime, so exact geometry,
  labels, and animation vary between runs — assert on _shape_ (SVG present,
  layers/bars/axes, palette, self-running loop), not pixel-exact output.
- This is the **minimal** cell: no host-side sandbox functions. The scene must
  loop or auto-advance on its own; do not expect interactivity.
- Behaviour comes from the OGUI runtime middleware + the frontend design skill,
  **not** a per-demo backend graph — the gateway only relays the model's
  `generateSandboxedUi` calls and events.
- Suggestion `message` strings double as deterministic aimock fixture keys
  (aligned with `showcase/aimock/d5-all.json`); keep them stable so pill clicks
  produce a stable tool call rather than a generic catch-all fixture.
