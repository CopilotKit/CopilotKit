<Variant for="built-in">
CopilotKit's default agent is built-in — connects directly to an LLM with tools and gen UI, no external framework needed. Switch the integration pill in the sidebar to see setup for LangChain, ADK, or any of the other supported backends.
</Variant>

<Variant for="langgraph">
This page wires CopilotKit up to a LangChain agent. You keep your LangChain code as-is and gain a chat UI, generative components from tool calls, two-way shared state with your graph, and `interrupt()` flows that resolve in React — without rewriting your agent. The runtime connects through the LangChain Platform / LangSmith deployment URL.
</Variant>

<Variant for="adk">
This page wires CopilotKit up to a Google ADK agent. Your ADK agent stays as a FastAPI service; CopilotKit adds a production-grade chat UI, generative UI for tool calls, and bidirectional state sync with `tool_context.state` — no agent rewrite needed.
</Variant>

<Variant for="a2a">
This page sets up CopilotKit with an A2A (Agent-to-Agent) backend using the with-a2a-a2ui starter template. A2A is an open protocol for building multi-agent systems where agents discover and call each other; CopilotKit gives you the user-facing layer — chat UI, tool calls rendered as React, and declarative A2UI for streaming agent-emitted layouts. The agent and frontend ship pre-wired.
</Variant>

<Variant for="ag2">
This page wires CopilotKit up to an AG2 (formerly AutoGen) backend. The AG2 agent runs as a FastAPI server; CopilotKit adds the chat UI, generative components, and human-in-the-loop flows on top — your AG2 logic doesn't change.
</Variant>

<Variant for="agent-spec">
This page wires CopilotKit up to an Agent Spec backend (a declarative agent config format), using the LangChain adapter as the primary supported runtime. CopilotKit gives your spec-driven agent a chat UI, generative UI, and human-in-the-loop without coding it from scratch.
</Variant>

<Variant for="agno">
This page sets up CopilotKit with an Agno agent. The Agno agent runs as a Python ASGI app exposed via the AG-UI interface; CopilotKit's runtime connects to it.
</Variant>

<Variant for="aws-strands">
This page sets up CopilotKit with an AWS Strands agent. Your Strands agent runs as a FastAPI server; CopilotKit's runtime connects to it over HTTP.
</Variant>

<Variant for="crewai-flows">
This page sets up CopilotKit with a CrewAI Flow deployed on CrewAI Enterprise, routed through Copilot Cloud — no self-hosted runtime needed.
</Variant>

<Variant for="llamaindex">
This page sets up CopilotKit with a LlamaIndex agent workflow. Your LlamaIndex agent runs as a FastAPI server; CopilotKit's runtime connects to it over HTTP.
</Variant>

<Variant for="mastra">
This page sets up CopilotKit with a Mastra agent. The Mastra agent runs as a separate dev server; CopilotKit's runtime connects to it.
</Variant>

<Variant for="microsoft-agent-framework">
This page sets up CopilotKit with the Microsoft Agent Framework (.NET or Python). Your MAF agent runs as a separate server; CopilotKit's runtime connects via HTTP.
</Variant>

<Variant for="pydantic-ai">
This page sets up CopilotKit with a Pydantic AI agent. Your Pydantic AI agent is exposed via `agent.to_ag_ui()` as a FastAPI app; CopilotKit's runtime connects to it.
</Variant>

## Setup

<Variant for="built-in">
<Steps>

<Step>

### Install packages

Create a new Next.js app and install the CopilotKit packages:

```bash
npx create-next-app@latest my-copilot-app
cd my-copilot-app
npm install @copilotkit/react-core @copilotkit/runtime
```

</Step>

<Step>

### Configure environment

Create a `.env` file in your project root and add your OpenAI API key:

```plaintext
OPENAI_API_KEY=your_openai_api_key
```

</Step>

<Step>

### Set up the runtime

Create an API route at `app/api/copilotkit/route.ts` with the `BuiltInAgent` and `CopilotRuntime`:

```ts
import {
  CopilotRuntime,
  copilotRuntimeNextJSAppRouterEndpoint,
} from "@copilotkit/runtime";
import { BuiltInAgent } from "@copilotkit/runtime/v2";
import { NextRequest } from "next/server";

const builtInAgent = new BuiltInAgent({
  model: "openai:gpt-5.4-mini",
});

const runtime = new CopilotRuntime({
  agents: { default: builtInAgent },
});

export const POST = async (req: NextRequest) => {
  const { handleRequest } = copilotRuntimeNextJSAppRouterEndpoint({
    runtime,
    endpoint: "/api/copilotkit",
  });

  return handleRequest(req);
};
```

</Step>

<Step>

### Add the chat UI

Wrap your app with the `CopilotKit` provider and add a chat UI component.

Update `app/layout.tsx`:

```tsx
"use client";
import { CopilotKit } from "@copilotkit/react-core/v2";
import "@copilotkit/react-core/v2/styles.css";
import './globals.css';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <CopilotKit runtimeUrl="/api/copilotkit" agent="default">
          {children}
        </CopilotKit>
      </body>
    </html>
  );
}
```

Then add the chat interface to `app/page.tsx`:

```tsx
import { CopilotSidebar } from "@copilotkit/react-core/v2";

export default function Page() {
  return (
    <main>
      <h1>Your App</h1>
      <CopilotSidebar />
    </main>
  );
}
```

Start your development server:

```bash
npm run dev
```

Navigate to `http://localhost:3000` — your copilot is live.

</Step>

</Steps>
</Variant>

```ts
import {
  CopilotRuntime,
  ExperimentalEmptyAdapter,
  copilotRuntimeNextJSAppRouterEndpoint,
} from "@copilotkit/runtime";
import { LangGraphAgent } from "@copilotkit/runtime/langgraph";
import { NextRequest } from "next/server";

const serviceAdapter = new ExperimentalEmptyAdapter();

const runtime = new CopilotRuntime({
  agents: {
    sample_agent: new LangGraphAgent({
      deploymentUrl: process.env.LANGGRAPH_DEPLOYMENT_URL || "http://localhost:8123",
      graphId: "sample_agent",
      langsmithApiKey: process.env.LANGSMITH_API_KEY || "",
    }),
  }
});

export const POST = async (req: NextRequest) => {
  const { handleRequest } = copilotRuntimeNextJSAppRouterEndpoint({
    runtime,
    serviceAdapter,
    endpoint: "/api/copilotkit",
  });

  return handleRequest(req);
};
```

Then start your LangChain agent server (from the agent directory):

```bash
npx @langchain/langgraph-cli dev --port 8123 --no-browser
```

<Step>

### Define your Agent Spec agent

Create a `main.py` file inside `ag-ui/integrations/agent-spec/python/ag_ui_agentspec/` that declares your agent and starts the FastAPI server:

```python title="ag_ui_agentspec/main.py"
from pyagentspec.agent import Agent
from pyagentspec.llms import OpenAiCompatibleConfig
from pyagentspec.serialization import AgentSpecSerializer
from fastapi import FastAPI
from ag_ui_agentspec.agent import AgentSpecAgent
from ag_ui_agentspec.endpoint import add_agentspec_fastapi_endpoint
import uvicorn

import os

agentspec_agent = Agent(
    name="AgentSpecAgent",
    description="A starter Agent that can call tools.",
    system_prompt="You are a helpful assistant.",
    llm_config=OpenAiCompatibleConfig(
        name="my-llm",
        model_id=os.getenv("OPENAI_MODEL", "gpt-5.4-mini"),
        url=os.getenv("OPENAI_BASE_URL", "https://api.openai.com/v1"),
    ),
)

agent_spec_config = AgentSpecSerializer().to_json(agentspec_agent)

# Or load your own Agent Spec config:
# agent_spec_config = <loaded json/yaml string of your Agent Spec agent>

runtime = "langgraph"  # or "wayflow"

app = FastAPI()
agent = AgentSpecAgent(agent_spec_config=agent_spec_config, runtime=runtime)
add_agentspec_fastapi_endpoint(app, agentspec_agent=agent, path="/")

if __name__ == "__main__":
    uvicorn.run("main:app", host="0.0.0.0", port=8000, reload=True)
```

Start the agent server:

```bash
uv run main.py
```

Your Agent Spec agent is now available at `http://localhost:8000`.

    ```python title="main.py"
                azure_endpoint=os.getenv("AZURE_OPENAI_ENDPOINT"),
            )

            return OpenAIChatClient(
                model=os.getenv("OPENAI_CHAT_MODEL_ID", "gpt-5.4-mini"),
                api_key=os.getenv("OPENAI_API_KEY"),
            )
        raise RuntimeError(
            "Set either AZURE_OPENAI_ENDPOINT + AZURE_OPENAI_API_KEY, or OPENAI_API_KEY."

        client=chat_client,

    app = FastAPI(title="Microsoft Agent Framework - Quickstart")
    ```

### Create a Copilot Runtime API route

CopilotKit requires a Copilot Runtime endpoint to communicate with your agent. In your Next.js app, create `app/api/copilotkit/route.ts`:

```tsx title="app/api/copilotkit/route.ts"
import {
  CopilotRuntime,
  ExperimentalEmptyAdapter,
  copilotRuntimeNextJSAppRouterEndpoint,
} from "@copilotkit/runtime";
import { HttpAgent } from "@ag-ui/client";
import { NextRequest } from "next/server";

// Use the empty adapter — we're routing directly to our AG-UI agent.
const serviceAdapter = new ExperimentalEmptyAdapter();

const runtime = new CopilotRuntime({
  agents: {
    my_agent: new HttpAgent({ url: "http://localhost:8000/" }),
  },
});

export const POST = async (req: NextRequest) => {
  const { handleRequest } = copilotRuntimeNextJSAppRouterEndpoint({
    runtime,
    serviceAdapter,
    endpoint: "/api/copilotkit",
  });

  return handleRequest(req);
};
```

## Connect a coding agent

Before you keep building, take a minute to wire up your coding agent — Cursor, Claude Code, Windsurf, Codex, anything that supports MCP or skills. Once connected, your agent has first-class knowledge of CopilotKit's v2 hooks, runtime config, and per-framework integration patterns. When you ask it to "add a frontend tool" or "wire up shared state", it reaches for the right APIs instead of guessing from training data.

There are two install paths and they compose — the **skill** teaches your coding agent the patterns and workflows; the **MCP server** lets it search live docs and source. Install both for the best experience.

### Install the skill

The skills are distributed via [`agentskills.io`](https://agentskills.io) and work across Claude Code, Codex, Cursor, OpenCode, and Gemini CLI. From your project root:

```bash
npx skills add copilotkit/skills --full-depth -y
```

Re-run the same command any time to pull the latest. This installs seven skills: `copilotkit-setup`, `copilotkit-develop`, `copilotkit-integrations`, `copilotkit-debug`, `copilotkit-upgrade`, `copilotkit-agui`, and `copilotkit-contribute`.

### Connect the MCP server

The hosted MCP server at `https://mcp.copilotkit.ai/mcp` exposes `search-docs` and `search-code` tools so your agent can look up the exact API signature, hook, or example it needs.

## Try generative UI

Your agent can render real React components — not just text. Register one with `useComponent` and the agent decides when to show it.

Add this to `app/page.tsx`:

```tsx
"use client";
import { CopilotSidebar, useComponent } from "@copilotkit/react-core/v2"; // [!code highlight]
import { z } from "zod";

export default function Page() {
  // [!code highlight:14]
  useComponent({
    name: "showWeather",
    description: "Display a weather card for a city.",
    parameters: z.object({
      city: z.string(),
      temperature: z.number(),
      condition: z.string(),
    }),
    render: ({ city, temperature, condition }) => (
      <div className="rounded-lg border p-4 my-2 bg-white shadow-sm">
        <h3 className="font-semibold">{city}</h3>
        <p className="text-2xl font-bold">{temperature}°F</p>
        <p className="text-sm text-gray-500">{condition}</p>
      </div>
    ),
  });

  return (
    <main>
      <h1>Your App</h1>
      <CopilotSidebar />
    </main>
  );
}
```

Now ask the chat **"What's the weather in San Francisco?"** — the agent calls `showWeather` with the right values and the card renders inline. That's generative UI in 14 lines.
