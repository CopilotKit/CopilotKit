Generative UI lets your agent render rich React components directly in the chat — cards, charts, forms, previews, anything. Instead of plain text replies, the agent picks the right component for the job and CopilotKit drops it inline.

## The mental model

There are four ways an agent can drive UI in CopilotKit. Each maps to one primitive:

<Columns cols={2}>
  <Card title="Render tool calls" icon="lucide/Eye" href="/generative-ui/tool-rendering">
    The agent calls one of your existing tools — you render a custom component for that call instead of plain JSON. **`useRenderTool`**
  </Card>
  <Card title="Components as tools" icon="lucide/Component" href="/generative-ui/your-components/display-only">
    Register a React component as a tool the agent can invoke. The agent decides when to show it; CopilotKit renders it with typed args. **`useComponent`**
  </Card>
  <Card title="Interactive components" icon="lucide/MessagesSquare" href="/generative-ui/your-components/interactive">
    The agent renders a component that asks the user a question and waits for a response. Approval flows, inline forms, confirmations. **`useHumanInTheLoop`**
  </Card>
  <Card title="Declarative UI (A2UI)" icon="lucide/Layout" href="/generative-ui/a2ui">
    The agent emits UI structure directly — no React components needed. CopilotKit renders the spec for you. Best for fully agent-driven layouts. **A2UI middleware**
  </Card>
</Columns>

## Which one should I use?

| If you want… | Use |
|---|---|
| To visualize what an existing tool just did | [Render tool calls](/generative-ui/tool-rendering) |
| The agent to decide when to show a card / chart / status | [Components as tools](/generative-ui/your-components/display-only) |
| The agent to ask the user something and wait for an answer | [Interactive components](/generative-ui/your-components/interactive) |
| Fully LLM-driven layouts, no per-tool React work | [Declarative UI](/generative-ui/a2ui) |
| UI generated by an MCP server | [UI from MCP servers](/generative-ui/mcp-apps) |

## A 30-second example

Register a React component as a tool — that's it. The agent now has a `showWeather` capability and can call it whenever it wants:

```tsx
import { useComponent } from "@copilotkit/react-core/v2";
import { z } from "zod";

useComponent({
  name: "showWeather",
  description: "Display a weather card for a city.",
  parameters: z.object({
    city: z.string(),
    temperature: z.number(),
    condition: z.string(),
  }),
  render: ({ city, temperature, condition }) => (
    <div className="rounded-lg border p-4">
      <h3 className="font-semibold">{city}</h3>
      <p className="text-2xl">{temperature}°F</p>
      <p className="text-sm text-gray-500">{condition}</p>
    </div>
  ),
});
```

Tell the agent "what's the weather in San Francisco?" and the card appears in the chat with the right values filled in.
