## Overview

CopilotKit V2 consolidates the frontend into a single package. Both hooks and UI components are now exported from `@copilotkit/react-core/v2`. Your backend does not need any changes.

**What's changing:**

| Before | After |
|--------|-------|
| `@copilotkit/react-core` | `@copilotkit/react-core/v2` |
| `@copilotkit/react-ui` | `@copilotkit/react-core/v2` |
| `@copilotkit/react-ui/styles.css` | `@copilotkit/react-core/v2/styles.css` |

**What's NOT changing:**
- Backend packages (`@copilotkit/runtime`, etc.) — no changes needed
- Your `CopilotRuntime` configuration — stays the same
- Agent definitions and backend setup — stays the same

## Hook Migration

The v2 API renames and restructures the core hooks. The table below maps every v1 hook to its v2 equivalent.

| v1 hook | v2 replacement | Notes |
|---|---|---|
| `useCopilotAction` | `useFrontendTool` (for tools) or `useHumanInTheLoop` (for HITL) | Actions become Tools in v2. Use `useFrontendTool` for side-effect tools; use `useHumanInTheLoop` when the tool needs to pause and wait for a user response. |
| `useCopilotReadable` | `useAgentContext` | Pass app state to the agent. `useAgentContext` accepts `{ description, value }` — same shape as `useCopilotReadable`, without `parentId` or `categories`. |
| `useCopilotChat` | `useAgent` | The lower-level chat-state hook is replaced by `useAgent`, which exposes `agent.messages`, `agent.isRunning`, and `agent.state` directly. |
| `useCoAgent` | `useAgent` | `useCoAgent` was the shared-state wrapper around the old agent protocol. `useAgent` replaces it: use `agent.state` and `agent.setState()` for bidirectional state, and `agent.runAgent()` / `copilotkit.runAgent()` to trigger runs. |
| `useCoAgentStateRender` | `useComponent` (visual component) or `useRenderTool` (progress/status render) | State-render hooks become tool renderers in v2. Use `useComponent` when you want the model to choose to display a named component; use `useRenderTool` to render while a specific tool call is in progress or complete. |
| `useMakeCopilotDocumentReadable` | No direct v2 equivalent | This hook registered document-typed context (PDF/text documents) in v1. In v2, pass document content as a string value via `useAgentContext`. The document-pointer abstraction does not exist in v2. |
| `useMakeCopilotReadable` | `useAgentContext` | Not found in the current source — was an early alias for `useCopilotReadable`. Migrate to `useAgentContext` the same way. |
| `useChatHistory` | `useThreads` | Thread/history management. `useThreads` exposes `threads`, `renameThread`, `archiveThread`, and `deleteThread` for the Intelligence platform. Note: `useChatHistory` is not found in the current codebase — you may be using a very early v1 API. |

<Note>
  The v1 hooks `useMakeCopilotReadable` and `useChatHistory` are not present in the current v1 source tree. If your code imports them, they likely came from a pre-release or community fork. Migrate to `useAgentContext` and `useThreads` respectively.
</Note>

## Migration Steps

<Steps>
<Step>
### Update `@copilotkit/react-core` imports

Replace imports from `@copilotkit/react-core` with `@copilotkit/react-core/v2`.

#### Before
```tsx
import { CopilotKit } from "@copilotkit/react-core";
import { useCopilotReadable, useCopilotAction } from "@copilotkit/react-core";
```

#### After
```tsx
import { CopilotKit } from "@copilotkit/react-core/v2";
import { useAgent } from "@copilotkit/react-core/v2";
```

</Step>
<Step>
### Replace `@copilotkit/react-ui` imports

UI components like `CopilotChat`, `CopilotSidebar`, and `CopilotPopup` are now exported from `@copilotkit/react-core/v2`.

#### Before
```tsx
import { CopilotPopup } from "@copilotkit/react-ui";
import { CopilotSidebar } from "@copilotkit/react-ui";
import { CopilotChat } from "@copilotkit/react-ui";
```

#### After
```tsx
import { CopilotPopup } from "@copilotkit/react-core/v2";
import { CopilotSidebar } from "@copilotkit/react-core/v2";
import { CopilotChat } from "@copilotkit/react-core/v2";
```

</Step>
<Step>
### Update your styles import

#### Before
```tsx
import "@copilotkit/react-ui/styles.css";
```

#### After
```tsx
import "@copilotkit/react-core/v2/styles.css";
```

</Step>
<Step>
### Upgrade `@ag-ui/client` (if using directly)

If you import from `@ag-ui/client` directly, upgrade to the latest version:

```bash
npm install @ag-ui/client@latest
```

Note: If you only use CopilotKit's React packages, `@ag-ui/client` types are already re-exported from `@copilotkit/react-core/v2` and you don't need a separate install.

</Step>
</Steps>

## Full Example

### Before

```tsx
import { CopilotKit } from "@copilotkit/react-core";
import { CopilotPopup } from "@copilotkit/react-ui";
import "@copilotkit/react-ui/styles.css";

export function App() {
  return (
    <CopilotKit runtimeUrl="/api/copilotkit">
      <YourApp />
      <CopilotPopup />
    </CopilotKit>
  );
}
```

### After

```tsx
import { CopilotKit, CopilotPopup } from "@copilotkit/react-core/v2";
import "@copilotkit/react-core/v2/styles.css";

export function App() {
  return (
    <CopilotKit runtimeUrl="/api/copilotkit">
      <YourApp />
      <CopilotPopup />
    </CopilotKit>
  );
}
```
