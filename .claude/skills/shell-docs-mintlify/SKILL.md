# shell-docs-mintlify

Mintlify Astro docs with a **config-driven IA** at `showcase/shell-docs-mintlify/`. One canonical
doc per topic, aliased to all integrations. Source of truth lives in
`showcase/shell-docs-mintlify/integrations.config.ts`. Read [`showcase/shell-docs-mintlify/CLAUDE.md`](../../showcase/shell-docs-mintlify/CLAUDE.md)
for the full reference — this file is the load-bearing summary.

## The 30-second mental model

- **URL = integration.** `/quickstart` is built-in. `/langgraph/quickstart` is LangGraph. The sidebar pill reflects the URL, never the other way around.
- **One file per topic.** A page like `frontend-tools.mdx` lives at `showcase/shell-docs-mintlify/docs/frontend-tools.mdx`. Aliases at `showcase/shell-docs-mintlify/docs/langgraph/frontend-tools.mdx` etc. are AUTO-GENERATED — never hand-edit them.
- **Per-integration code differences** go inside `<Variant for="<slug>">...</Variant>` blocks in the canonical file. CSS swaps which block renders based on `body[data-integration]`.
- **`integrations.config.ts` is the IA.** Adding/renaming/moving pages = editing this one file, then `npm run gen`.
- **v2 only.** Never document v1 hooks, components, service adapters. The directive is firm.

## Common task → where to edit

| Task | Edit | Then run |
|---|---|---|
| Add a universal page (concept) | `integrations.config.ts` (`universalPages` array) + create `docs/<slug>.mdx` | `npm run gen` |
| Rename a page in the sidebar | `integrations.config.ts` `title:` field | `npm run gen` |
| Move a page to a different sidebar group | `integrations.config.ts` `group:` field | `npm run gen` |
| Add a new integration | `integrations.config.ts` `integrations` array | `npm run gen` |
| Add an integration-only page | `integrations.config.ts` `integrationOnlyPages` + create `docs/<integration>/<slug>.mdx` | `npm run gen` |
| Per-integration code differs in one section | Add `<Variant for="<slug>">` blocks in the canonical `docs/<slug>.mdx` | (no regen needed) |
| Embed a live showcase demo | Add `<ShowcaseDemo feature="<feature>" />` in any MDX | (no regen needed; URLs come from `showcase.config.json`) |

## Generated files — DON'T edit

These regenerate from config every `npm run gen` (and automatically on `predev` / `prebuild`):

- `showcase/shell-docs-mintlify/docs/docs.json` — Mintlify navigation
- `showcase/shell-docs-mintlify/docs/<integration>/<slug>.mdx` — alias stubs (have `# AUTO-GENERATED` in YAML frontmatter)
- `showcase/shell-docs-mintlify/src/styles/integrations.css` — per-integration visibility rules
- `showcase/shell-docs-mintlify/src/lib/showcase.config.json` — showcase URL mapping

If a file has `# AUTO-GENERATED by scripts/generate-routing.ts` in its frontmatter, edit the source (`docs/<canonical-slug>.mdx`) instead.

## Hard rules

1. **v2 only.** APIs from `@copilotkit/react-core/v2`, `@copilotkit/runtime/v2`. The v1 hooks (`useCopilotAction`, `useCopilotReadable`) and v1 service adapters (`AnthropicAdapter`, etc.) are deprecated. Documenting them is out of scope.
2. **Model strings are real.** `gpt-5.4-mini`, `gpt-5.4`, etc. are valid product model identifiers. **Never auto-correct** to `gpt-4o` or `gpt-4.1` no matter how typo-like they look.
3. **Use `<Columns cols={N}>` not `<Cards>`.** Mintlify renders `<Cards>` inconsistently; the project uses `<Columns>` as the canonical wrapper.
4. **Use `<AccordionGroup>` not `<Accordions>`.** Mintlify's accordion API.
5. **Don't fabricate API surface.** When porting docs from the legacy Fumadocs source (`docs/content/docs/integrations/built-in-agent/`), verify imports/types against the actual v2 source files.

## Component vocabulary in MDX

Auto-imported by Mintlify (no import statement needed):
`<Note>`, `<Tip>`, `<Info>`, `<Warning>`, `<Card>`, `<Columns>`, `<Tabs>`/`<Tab>`,
`<Steps>`/`<Step>`, `<AccordionGroup>`/`<Accordion>`.

Custom components registered globally in `showcase/shell-docs-mintlify/src/pages/[...slug].astro`:
`<Variant for="<slug>">`, `<ShowcaseDemo feature="<key>" />`.

## How to start the preview

```bash
# Dev server with HMR
cd showcase/shell-docs-mintlify && npm run dev          # http://localhost:4321
```

Or via Claude Preview MCP:
```
mcp__Claude_Preview__preview_start { name: "shell-docs-mintlify" }
```

The launch config in `.claude/launch.json` already wires `shell-docs-mintlify` as a named
preview, so the second form just works.

## How to verify before declaring done

```bash
cd showcase/shell-docs-mintlify
npm run gen        # regen all generated artifacts
npm run build      # full static build (catches missing canonical files, broken nav)
npm test           # Vitest unit tests for lib/integration.ts and the components
```

A clean run produces ~700+ HTML pages (universal × integrations + reference + integration-only). If the page count drops unexpectedly, an alias file failed to render and the build will tell you which canonical it was looking for.

## Common gotchas

- **Adding a page to `universalPages` without creating `docs/<slug>.mdx`** → build fails because the alias generator points to a non-existent canonical. Always create the .mdx file at the same time as the config entry.
- **Editing an alias file** → silently overwritten on next `npm run gen`. The marker `# AUTO-GENERATED by scripts/generate-routing.ts` in the frontmatter is the tell.
- **Forgetting to clear caches after big config changes** → Mintlify holds compiled component references. `rm -rf .mintlify .astro dist node_modules/.vite` if rebuild errors look stale.
- **Premium content** → removed in this branch. Don't add premium pages back without explicit user approval.
